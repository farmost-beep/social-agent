# 社交关系AI管家

一个本地运行的AI管家，自动追踪所有社交关系中「最近干了什么」和「下一步该干什么」。

## 版本：v3.0.0（Social-CLI）

v3.0 起，本 skill **不再强依赖 Claude Code**。所有功能也支持独立 CLI 运行（`social` 命令）。详见 [README.md](../../README.md) 和 [docs/MIGRATION.md](docs/MIGRATION.md)。

## 前置条件

- Python 3.9+
- ~~Claude Code~~（v3.0 起可选，不再必需）
- 微信推送：cc-connect（可选）

## 触发场景

用户提到"社交关系"、"关系管家"、"帮我记住"、"联系人"、"记一下"、"拟消息"、"社交"、"人情世故"、"好久没联系"、"该联系谁"等与人际关系管理相关的词句时触发。

## 核心规则

1. 涉及联系人数据的查询，**必须读 `contacts.json` 真实数据**，禁止从对话上下文猜答案。
2. 数据修改用 `src/social.py` 命令，不要直接编辑 JSON。
3. **删除或取消任何待办事项，必须由用户本人明确确认**，AI不得自行将待办标记为completed或删除。即使用户提到"某某事项不用跟了"，也必须追问"确认要取消吗？"得到肯定答复后再操作。
4. **记录取消原因**：取消待办时，必须在待办中记录取消原因（`note`字段），以便日后追溯。"谁在什么时候为什么取消"要可查。
5. **时间冲突不自动取消**：两个活动时间冲突时，**同时保留两条待办**，标注时间冲突提醒即可，不自动取消其中任何一条。由用户自己决定如何处理。
6. **日程提前1小时提醒机制**：对于有时间标注的日程，自动设置提前1小时的提醒推送（通过 CronCreate 设一次性提醒）。每日晨间汇总推送（7:33）时检查当天所有日程，为缺少提前提醒的日程补设。
7. **优先使用昵称/别称**：如果联系人有昵称或别称，在待办、时间线、推送中优先使用昵称。例如"姜知清"用"姜少"。

## 交互流程

### 1. 记录互动
```
用户：记一下：和张总聊了项目进展
→ 自动查找/创建联系人 → 记录时间线 → 提取待办
```

### 2. 查询联系人状态
```
用户：张总最近咋样
→ 查询时间线 → 返回最近互动+待办
```

### 3. AI拟稿
```
用户：给张总拟条消息
→ 提取上下文 → 生成草稿 → 确认→发送
```

### 4. 检查需跟进的关系
```
用户：最近该联系谁
→ 分析所有关系 → 列出14天+未联系的冷却关系
```

### 5. 查看待办
```
用户：最近有啥要跟进的
→ 列出P0/P1优先待办
```

## 子命令

### v3.0 新统一命令（推荐）

`social` 是 v3.0 起的统一 CLI 入口（pip install -e . 后全局可用）：

| 命令 | 说明 |
|:----|:-----|
| `social status` | 联系人状态 |
| `social todos` | 待办列表 |
| `social draft -m "上下文"` | AI 拟稿 |
| `social chat "msg"` | 直接对话 |
| `social config show` | 查看 LLM 配置 |
| `social version` | 版本信息 |

### v2 兼容命令（仍可用）

| 命令 | 说明 |
|:----|:-----|
| `python3 src/social.py status` | 查看所有联系人 |
| `python3 src/social.py todos` | 查看待办 |
| `python3 src/social.py dashboard` | 仪表盘 |
| `python3 src/social.py adjust` | 强度调整建议 |
| `python3 src/social.py birthdays` | 生日提醒 |
| `python3 src/agent.py --chat "消息"` | 自然语言交互 |

## 数据目录

所有数据存储在 `data/` 目录下（可配置），JSON格式，本地存储不上云：

```
data/
├── contacts.json      # 联系人库
├── timeline.json      # 时间线
└── todos.json         # 待办队列
```
