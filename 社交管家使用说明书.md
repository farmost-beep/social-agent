## 1. 快速开始

```bash
# 1. 查看所有联系人
cd ~/.claude/skills/social-agent
python3 social.py dashboard

# 2. 添加第一个联系人
python3 social.py add-contact zhangzong 张总 --role 客户

# 3. 记录互动
python3 social.py log zhangzong "聊了科技金融产品方案，有兴趣进一步了解"

# 4. 查看待办
python3 social.py todos

# 5. 设置别名（以后可以用昵称）
python3 agent.py --chat "张总是张总"
```

## 2. 安装与部署

### 2.1 前置条件

| 依赖 | 说明 | 是否必需 |
|:----|:-----|:--------|
| Python 3.9+ | 运行时 | ✅ 必需 |
| Claude Code | AI拟稿引擎（`claude` 命令可用） | ✅ 必需 |
| wechat-claude-code 桥接 | 微信推送通道 | ⭕ 可选 |
| matplotlib + networkx | 关系图谱生成 | ⭕ 可选 |

### 2.2 安装步骤

#### 方式一：通过 Skills CLI 安装（推荐）

```bash
# 全局安装 skill
npx skills add social-agent -g -y

# 验证安装
npx skills list | grep social-agent
```

#### 方式二：手动克隆

```bash
git clone https://github.com/farmost-beep/social-agent.git
# 将项目链接到 Claude Code skill 目录
ln -s "$PWD/social-agent" ~/.claude/skills/social-agent
```

#### 方式三：从已有仓库更新

```bash
cd ~/.claude/skills/social-agent
git pull origin main
```

### 2.3 Claude Code 加载 Skill

本 Skill 会自动注册到 Claude Code 的可用命令列表中。加载机制由 `~/.claude/skills/social-agent/SKILL.md` 定义：

- 当用户提到"社交关系""关系管家""帮我记住""联系人""记一下""拟消息""该联系谁"等关键词时自动触发
- 也可直接通过 Claude 中说指令交互（无需手动加载）

## 3. 数据导入

### 3.1 从本地文件批量导入

项目提供了 `import_contacts.py` 脚本，自动扫描本地 Excel 数据源：

```bash
cd ~/.claude/skills/social-agent
python3 import_contacts.py
```

脚本自动读取以下数据源：

| 数据源 | 路径 | 说明 |
|:------|:-----|:-----|
| 校友联系方式 | `deliverables/career/校友联系方式.xls` | 科大校友总表 |
| 校友联系方式2 | `deliverables/career/校友联系方式2.xls` | 补充表 |
| 校友活动终表 | `deliverables/career/校友活动终表.xls` | 活动报名表 |
| VCF通讯录 | `deliverables/.../*.vcf` | 微信导出通讯录 |

**导入逻辑：**

1. 按姓名去重（模糊匹配，含英文名+中文名组合）
2. 自动分类：同门（九系/精密机械）、同行（金融/银行/证券/基金）、创业、校友
3. 有微信联系方式的强度+1
4. 生成时间线记录

**⚠️ 注意事项：**

- 同门判定只看 `dept` 和 `company` 字段，不看 `grade`（避免"09级"误判为"九系"）
- 导入前脚本会自动备份现有联系人
- 重复联系人不会覆盖，仅补充微信ID等缺失字段

### 3.2 手动添加联系人

```bash
# 添加客户
python3 social.py add-contact li_zong 李总 --role 客户

# 添加朋友（设别名方便查找）
python3 social.py add-contact wang_ge 王晓明 --role 朋友
python3 agent.py --chat "王哥是王晓明"

# 添加同门（带标签）
python3 social.py add-contact liu_bojian 刘博蹇 --role 同门
python3 agent.py --chat "刘博蹇的标签是九系 0701"
```

## 4. 交互方式

### 4.1 在 Claude Code 对话中直接说（推荐日常使用）

| 你说 | 它做 |
|:----|:-----|
| `记一下：和张总聊了项目合作` | 记录互动，自动提取待办 |
| `张总最近咋样` | 查询最近互动记录 + 待办 |
| `最近该联系谁` | 列出14天+未联系的关系 |
| `最近有啥要跟进的` | 列出待办事项 |
| `给王哥拟条消息` | AI生成消息草稿 |
| `给王哥拟条消息，正式一点` | 带语气调节的AI拟稿 |
| `有多少联系人` | 实时统计 |
| `李哥是李总` | 设别名 |
| `帮我把信哥删了` | 删除联系人 |

### 4.2 在微信里跟机器人聊（最方便）

需要对接 `wechat-claude-code` 桥接服务，桥接启动后：

```
→ 在微信里直接发消息给机器人
← 机器人自动回复查询结果或消息草稿
```

支持同上所有指令，且**设置好对方微信ID后，可以直发消息到对方微信**。

### 4.3 命令行操作

```bash
cd ~/.claude/skills/social-agent

# 全局仪表盘
python3 social.py dashboard

# 查看特定联系人
python3 social.py status 张总

# 查看所有联系人的大致状态
python3 social.py status

# AI交互模式
python3 agent.py --chat "最近该联系谁"

# 手动触发早间推送
python3 agent.py
```

## 5. 命令参考

### 5.1 `social.py` CLI 命令

| 命令 | 参数 | 说明 |
|:----|:-----|:-----|
| `dashboard` | 无 | 全局仪表盘：联系人总数、角色分布、待办数、冷却关系 |
| `status` | `[contact]` | 查看所有联系人或指定联系人的时间线 |
| `add-contact` | `<id> <name> --role <角色>` | 添加新联系人 |
| `log` | `<contact> <摘要>` | 记录互动，自动提取待办和关键信息 |
| `todos` | `[--priority P0]` | 查看待办（按优先级和截止日期排序） |
| `done` | `<todo-id>` | 完成待办 |
| `draft` | `<contact> [--tone 亲切/正式/简洁]` | AI生成消息草稿 |
| `send` | `<contact> [--tone 亲切/正式/简洁]` | 拟稿并推送到微信 |
| `check` | 无 | 检查所有14天+未联系的关系 |

### 5.2 `agent.py` 模式

| 命令 | 说明 |
|:----|:-----|

| `agent.py --chat "消息"` | 自然语言处理一条用户指令 |
| `agent.py --morning` | 手动推送早间概览（待办+冷却关系） |
| `agent.py --afternoon` | 手动推送午后建议（冷却关系列表） |
| `agent.py --evening` | 手动推送晚间回顾（今日互动总结） |

### 5.3 辅助脚本

| 脚本 | 命令 | 说明 |
|:----|:-----|:-----|

## 6. 自动化推送

### 6.1 推送时段（建议配置 crontab）

| 时段 | 内容 | 说明 |
|:---|:-----|:-----|
| ☀️ 09:00 | 早间概览 | 今日待办 + 冷却关系提醒 + 超期预警 |
| 💡 14:00 | 午后建议 | 列出14天+未联系的关系 |
| 🌙 21:00 | 晚间回顾 | 今日互动总结 + 待办状态 |
| ⏰ 每30分钟 | 紧急提醒 | 仅P0超期待办推送（需要daemon模式） |

### 6.2 crontab 配置示例

```bash
crontab -e
```

```cron
# 早间概览
0 9 * * * cd ~/.claude/skills/social-agent && python3 agent.py

# 午后建议
0 14 * * * cd ~/.claude/skills/social-agent && python3 agent.py --afternoon

# 晚间回顾
0 21 * * * cd ~/.claude/skills/social-agent && python3 agent.py --evening
```

### 6.3 守护进程模式（持续运行）

```bash
python3 agent.py --daemon &
```

每小时自动检查并推送提醒，仅超期P0待办才推送。

## 7. 高级功能

### 7.1 别名系统

不用记全名或ID，随时设别名：

```bash
# Claude对话中
"李哥是李总"
"王哥是王晓明"

# 或者命令行
python3 agent.py --chat "李哥是李总"
```

之后说"李哥"就能找到对应的联系人。

系统预置部分别名（来自微信昵称映射），也支持随时动态添加。

### 7.2 微信直发

设置联系人的微信内部ID后，消息可以直接推送到对方微信：

```bash
python3 agent.py --chat "王哥的微信ID是wxid_xxxxxxxxxxxx@im.wechat"
```

之后：
```
"给王哥发消息：周末一起吃饭"
→ 拟稿 → 确认 → 直发对方微信
```

（内部ID需对方给机器人发过消息后从日志提取，或者通过微信Web端获取）

### 7.3 AI拟稿语气调节

支持三种语气：

| 语气 | 说明 | 适用场景 |
|:----|:-----|:--------|
| `亲切`（默认） | 朋友聊天风格，用"你" | 日常社交 |
| `正式` | 专业正式，用"您" | 商务客户 |
| `简洁` | 不超过30字，直说重点 | 熟人/快速确认 |

```bash
python3 social.py draft 李总 --tone 正式
```

拟稿规则：
- 不使用对方全名（微信对话本身就知道是谁）
- 如果必须称呼，用尊称（x总/x老师/x兄）
- 跟进事项语气不要太急
- 长度20-80字

### 7.4 待办自动管理

记录互动时，系统自动提取待办事项，按优先级和截止日期排序：

- **P0优先级**：含"投资""融资""条款""DD""引荐""会面""签约"等关键词
  - P0默认3天截止
- **P1优先级**：普通待办
  - P1默认7天截止
- 超期待办自动标红提醒

### 7.5 关系图谱

生成可视化的社交关系网络图：

```bash
python3 generate_graph.py
# 生成 data/relationship_graph.png
```

图谱按关系分类着色（同门/同行/创业/校友），节点大小代表关系强度。

### 7.6 Web管理界面

启动一个简单的Web界面管理联系人：

```bash
python3 web.py
# 默认 http://localhost:8080
```

支持：查看联系人列表、时间线、待办，添加/编辑联系人。

## 9. 常见问题

### Q: 数据安全吗？会上传到云吗？

所有数据存储在 `data/` 目录下的JSON文件，**纯本地存储，不上云**。备份只需备份整个 `data/` 目录。

### Q: 为什么查询数据时不直接跑脚本？

当用户通过微信询问时，脚本跑Bash命令会触发Claude Code的权限弹窗，影响体验。所以 SKILL.md 规定查询数据直接读JSON文件，不跑脚本。

### Q: 如何备份和恢复？

```bash
# 备份
cp -r ~/.claude/skills/social-agent/data ~/backups/social-agent-$(date +%Y%m%d)

# 恢复（覆盖）
cp ~/backups/social-agent-20260613/* ~/.claude/skills/social-agent/data/
```

### Q: 导入时联系人被误分类了怎么办？

分类逻辑在 `import_contacts.py` 的 `classify_contact()` 函数中。手动编辑 `contacts.json` 修改 `relation`、`sub_relation`、`tags` 字段即可。修改后重新运行 `social.py dashboard` 确认统计正确。

### Q: 如何对接微信推送？

配置 `wechat-claude-code` 桥接服务（参见该项目文档）。桥接启动后，`push.py` 自动通过WebSocket发送消息到微信。

### Q: 没有微信桥接，还能用吗？

能。所有核心功能（查询、记录、拟稿、待办）不需要微信桥接，在Claude Code对话中直接使用即可。

### Q: 联系人别名和微信ID重复了怎么办？

别名和微信ID是两套独立系统。别名用于在Claude对话中快速索引联系人，微信ID用于直发消息。两者不冲突。

### Q: 为什么"同门"分类经常出错？

同门判定只看 `dept` 和 `company` 字段是否含"九系"/"9系"/"精密机械"。如果某校友的实际系别没有在这些字段中体现，请手动在 `contacts.json` 中修改 `relation` 和 `tags`。
